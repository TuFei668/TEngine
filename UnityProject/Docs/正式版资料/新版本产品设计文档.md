# WordSearch 英语 — 完整产品设计文档

> 基于 Word Search Explorer 玩法改造，面向中国中小学生的英语词汇学习工具。

---

## 一、产品概要

| 维度 | 说明 |
|------|------|
| 产品名称 | WordSearch 英语（暂定） |
| 技术栈 | Unity → 微信小游戏 |
| 目标用户 | 中国中小学生（核心），成人英语学习者（后续） |
| 产品定位 | 轻度游戏 × 英语词汇学习 × 日常习惯养成工具 |
| 核心体验 | 30-60秒一关的找词游戏，可选学习模式，省份集体学习排行 |
| 商业模式 | 免费+订阅为主，弱广告（激励视频），不强制插屏 |

---

## 二、核心设计原则

1. **极简操作** → 5秒上手，滑动即玩
2. **无失败** → 没有Game Over，消除一切挫败感
3. **快反馈** → 每10秒一个正反馈
4. **短单局** → 30-60秒/关
5. **学习无感化** → 游戏模式下学习是隐性的，不打断节奏
6. **记忆曲线驱动** → 关卡词汇编排基于艾宾浩斯间隔重复，玩家在"正常玩关卡"的过程中自然完成科学复习，无需额外测试或复习环节
7. **集体感** → 个人行为汇聚为城市/省份集体数据
8. **防沉迷** → 面向未成年人，必须有时长限制和护眼提醒

## 四、核心玩法设计

### 4.1 基础操作流程

```
进入关卡 → 显示主题名 → 显示字母网格 + 目标词列表
→ 玩家手指按住字母沿直线拖动 → 彩色高亮线条跟随手指
→ 松手判定 → 正确：单词保留高亮，目标列表标记
           → 错误：高亮消失，无惩罚
→ 全部找到 → 结算（网格旋转动画）→ 下一关
```

### 4.2 单词方向规则

支持8个直线方向，只能直线不能拐弯：
- 横向：左→右、右→左
- 纵向：上→下、下→上
- 斜向：4个对角方向

难度递进中方向引入顺序：
1. 初期：仅横向
2. 中期：加入纵向、斜向
3. 后期：加入反向

### 4.3 目标词列表

- 位置：网格上方
- 布局：瀑布流排列，两行
- 内容：显示英文单词本身
- 找到后：标记已完成（划线/变色）

### 4.4 Bonus Word 机制

- 玩家滑出非目标但合法的英文单词 → 触发 Bonus Word
- 接受英语词典中几乎所有单词，但**专有名词和大写词不算**
- 攻击性词汇、缩写、前缀、后缀等也会被过滤
- 累计一定数量 → 充满 Bonus Meter（进度条）→ 奖励道具
- **Bonus Word 不计入学习积分**，只有目标词才计入
- 设计意图：鼓励多尝试，增加交互频次

### 4.5 Hidden Words 机制（拓展词汇）

- 部分关卡包含设计者预设的"隐藏词"
- 隐藏词**不在目标词列表中显示**，是"谜题词"
- 玩家偶然找到后奖励额外金币
- 与 Bonus Word 的区别：Hidden Words 是特定预设词，Bonus Words 是任何合法词
- **教育化改造：** 隐藏词可以设计为课外拓展词汇（与主题相关但不在课本要求内）
- 学习模式下找到隐藏词也会显示释义+发音

### 4.6 两种模式

**游戏模式（默认）**
- 找到单词 → 彩色高亮 + 音效 → 继续找
- 不弹出释义，不打断节奏
- 节奏目标：30-60秒/关

**学习模式（设置开关）**
- 找到单词 → 彩色高亮 + 底部浮现学习卡片（释义 + TTS发音按钮）
- 卡片自动显示2秒后淡出，不阻断操作
- 可点击卡片展开看例句
- 过关后额外显示单词回顾列表
- 节奏目标：60-90秒/关

## 五、关卡系统设计

### 5.1 学段系统与关卡包组织

**学段选择：**

玩家首次进入时选择学段，决定关卡起点和解锁路径：

| 选择学段 | 起点 | 解锁路径 |
|----------|------|----------|
| 小学 | 小学3年级上册 | 小学全部 → 解锁初中 → 解锁高中 → 解锁成人 |
| 初中 | 初中7年级上册 | 初中全部 → 解锁高中 → 解锁成人 |
| 高中 | 高中必修1 | 高中全部 → 解锁成人 |
| 成人 | 四级核心词 | 四级 → 六级 → 考研 → 雅思 |

**关卡包体系：**

关卡按学段→主题组织为"关卡包"，每个关卡包对应地图上的一个"地区"：

```
关卡包体系
├── 小学（primary）~800词
│   ├── primary_self_intro（自我介绍）20关
│   ├── primary_family（家庭）20关
│   ├── primary_animals（动物）20关
│   ├── primary_colors（颜色）20关
│   ├── primary_food（食物）20关
│   ├── primary_school（学校）20关
│   ├── primary_body（身体）20关
│   ├── primary_numbers（数字）20关
│   └── ...（按主题持续扩展）
├── 初中（junior_high）~1600词
│   ├── junior_daily_life（日常生活）20关
│   ├── junior_travel（旅行）20关
│   ├── junior_technology（科技）20关
│   ├── junior_environment（环境）20关
│   └── ...
├── 高中（senior_high）~3500词
│   ├── senior_society（社会）20关
│   ├── senior_science（科学）20关
│   ├── senior_economy（经济）20关
│   └── ...
└── 成人（adult）
    ├── cet4（四级核心词）
    ├── cet6（六级核心词）
    ├── postgrad（考研词汇）
    └── ielts（雅思词汇）
```

**展示关卡号 vs 实际关卡ID：**

所有玩家看到的界面都是从"关卡1"开始，线性往上推进。区别只是背后对应的关卡包不同：

```
高中生选了"高中"：
  界面显示：关卡1 → 关卡2 → ... → 关卡12 → 关卡13 → ...
  背后实际：pep_required1/unit1/level1 → pep_required1/unit1/level2 → ...

小学生选了"小学"：
  界面显示：关卡1 → 关卡2 → ... → 关卡12 → 关卡13 → ...
  背后实际：pep_grade3_s1/unit1/level1 → pep_grade3_s1/unit1/level2 → ...
```

主界面大按钮显示 **"Play Level 35"**，和原版一模一样。玩家不需要知道背后指向哪个关卡包。

**学段切换时的关卡衔接：**

小学生打完小学全部关卡（假设共360关），解锁初中时：
- 关卡360（小学最后一关）→ 关卡361（初中7年级上册 Unit1 第1关）
- 界面上关卡号一直递增，不会重置回1
- 地图上用视觉分区标识学段：`[小学3年级上] → ... → [小学6年级下] → 🔒初中`

### 5.2 内容来源

单词来源不按单一教材版本区分，而是将各版本教材（人教版/外研版/沪教版/北师大版等）的词汇汇总，按学段整理为统一词库。这样覆盖面更广，用户无需选择教材版本。

**学段词库体系：**

| 学段 | 词库来源 | 预估词汇量 | 主题化分组方式 |
|------|----------|-----------|---------------|
| 小学（3-6年级） | 人教版PEP + 外研版 + 沪教版 + 北师大版，取并集 | ~800词 | 按主题分组（动物、颜色、食物、家庭、学校、身体、数字等） |
| 初中（7-9年级） | 人教版 + 外研版 + 沪教版，取并集 | ~1600词 | 按主题分组（日常生活、旅行、科技、环境、文化等） |
| 高中（必修+选修） | 人教版 + 外研版 + 北师大版，取并集 | ~3500词 | 按主题分组（社会、科学、经济、哲学、文学等） |
| 成人 | 四级核心词 / 六级核心词 / 考研词汇 / 雅思词汇 | 各2000-5000词 | 按考试大纲+主题分组 |

**汇总规则：**
- 多版本教材取并集：只要任一版本教材包含该词，就纳入该学段词库
- 去重：同一个词在不同教材中出现只保留一次
- 按主题重新分组：不再按"Unit 1/Unit 2"这种教材章节组织，而是按语义主题（动物、食物、家庭...）分组，每个主题对应一个关卡包
- 主题化分组的好处：和原版 Word Search Explorer 的主题关卡（BIRD/FOOD/TRAVEL）设计一致，降低搜索难度，提供新鲜感

**MVP：** 小学词库中的1个主题（如"自我介绍"），约20个词，设计12关

### 5.2 记忆曲线驱动的词汇编排

关卡词汇编排基于艾宾浩斯间隔重复原理，将"科学复习"融入关卡推进本身。玩家感知到的只是"又一关找词"，不知道自己正在被科学地安排复习。

**为什么不用结算测试题/生词本？**
- 测试题会打断"找词→反馈→下一关"的心流循环，与原版"无失败、无阻断"的设计哲学矛盾
- 生词本引入"你答错了"的负面信号，与"只有正反馈"的设计冲突
- 在网格中找词本身就是"主动识别+视觉扫描+手指操作"的多通道编码，记忆效果优于被动测试题

**间隔复现规则：**

| 复现次数 | 间隔关卡数 | 对应记忆阶段 |
|----------|------------|-------------|
| 第1次复现 | 1-2关后 | 短期巩固 |
| 第2次复现 | 3-5关后 | 中期强化 |
| 第3次复现 | 6-10关后 | 长期固化 |
| 第4次复现 | 12-18关后 | 深度固化 |
| 主题总关 | 主题末尾（第20关） | 全量检验 |

**关键设计点：**
- 20关容纳约30个词（每关引入2-3个新词，前12关引入完毕，后8关纯复现+少量新词）
- 每7关一个难度循环，20关包含约3个完整循环（第1-7关、第8-14关、第15-20关）
- 第6/7关、第13/14关、第18/19关是三组"呼吸+奖励"关，不引入新词，降低认知负荷
- 第10关和第20关是景区卡片解锁节点，给玩家中期和终期的收集奖励
- 复现间隔从1关逐步拉大到12-18关，覆盖短期→中期→长期→深度四个记忆阶段
- 第20关（主题总关）包含全部词汇，作为最终的全量检验
- 后台埋点 `word_find_time` 可用于识别"困难词"，后续版本可据此动态调整复现频率

### 5.3 关卡节奏

每7关一个循环，20关包含约3个循环：
```
循环1（关卡1-7）：简单 → 简单 → 中等 → 简单 → 中等 → 呼吸关 → 奖励关
循环2（关卡8-14）：中等 → 中等 → 较难 → 中等 → 较难 → 呼吸关 → 奖励关
循环3（关卡15-20）：中等 → 较难 → 中等 → 呼吸关 → 准总关 → 总关
```

网格大小交替变化，形成"呼吸感"。难度随循环递进，但每个循环内部有起伏。

## 六、网格生成逻辑

### 6.1 生成流程

1. 确定网格尺寸
2. 从关卡单词池取出目标单词
3. 按随机顺序放入网格（随机起始位置+受难度限制的方向）
4. 检查冲突，冲突则重试
5. 空位填充随机字母
6. 干扰字母密度随难度递增（30% → 60%）

### 6.2 难度参数

| 参数 | 简单 | 中等 | 困难 |
|------|------|------|------|
| 网格大小 | 6×6 | 7×7 ~ 8×8 | 10×10+ |
| 单词数量 | 3-4 | 5-6 | 7-10 |
| 单词长度 | 3-4字母 | 4-6字母 | 6-8字母 |
| 方向 | 仅横向 | 横+竖+斜 | 全方向含反向 |
| 干扰字母 | 少(30%) | 中(45%) | 多(60%) |

## 七、道具系统

### 7.1 道具栏布局（底部固定栏，从左到右6个位置）

游戏界面底部固定显示道具栏，1:1复刻原版布局：

| 位置 | 道具 | 图标 | 价格 | 功能 | 设计意图 |
|------|------|------|------|------|----------|
| 1 | Bonus Words | 💬语音气泡 | 免费 | 查看当前关卡已找到的Bonus Words列表和进度条 | 信息查看，非消耗品。进度条接近满时制造"再找一个就能领奖"的驱动力 |
| 2 | Free Hint | ✅勾选 | FREE（看广告） | 观看激励视频，免费获得一次Normal Hint提示 | 广告入口，降低道具使用门槛。玩家不花金币也能获得帮助 |
| 3 | Get Coins | ▶️播放 | GET（看广告） | 观看激励视频，获取金币 | 核心广告变现入口。金币不足时的直接解决方案 |
| 4 | Wind Hint | ⚡闪电 | 200金币 | 移除网格中所有不属于目标单词的干扰字母，只留下目标词的字母 | 重度道具，大幅降低难度。定价200金币（通关约10金币/关），需约20关收入才能买一次，刻意制造金币饥渴 |
| 5 | Normal Hint | 💡灯泡 | 20金币 | 揭示一个目标单词的**起始字母位置**（高亮闪烁） | 轻度提示，常规消耗。角标显示剩余可用次数，制造紧迫感 |
| 6 | Rotate | 🔄旋转 | 15金币 | 整个网格旋转180度，字母位置不变，只换视角 | 不改变内容，给卡住的玩家一个新的视觉角度。原来从左到右的词变成从右到左 |

**道具栏设计要点：**
- 位置1（Bonus Words）是信息查看，不是消耗品，点击弹出当前关卡的Bonus Words列表和进度条
- 位置2-3（Free Hint / Get Coins）是**激励视频广告的直接入口**，标注"FREE"和"GET"引导点击
- 位置4-6（Wind Hint / Normal Hint / Rotate）是**金币消耗道具**，金币不足时引导玩家看广告或去商城
- Normal Hint 右上角有红色数量角标（如"1"），显示剩余可用次数
- Daily Challenge 界面不显示 Bonus Words 图标（因为每日挑战不支持Bonus Words机制）
- 道具使用后可能有限时Buff效果（如"x2 00:03"倒计时），Buff到期前在道具栏显示倒计时

**Bonus Meter（进度条）机制：**
- 进度条跨关卡累计，不会单关重置
- 累计找到一定数量Bonus Words → 进度条填满 → 奖励25金币
- 填满后提供两个领取选项："Claim"直接领取25金币，或"▶ Double Coins"看广告领取50金币（双倍）
- 这是激励视频广告的又一个植入点

### 7.2 道具经济闭环

```
玩家卡住 → 想用道具
  ├── 有金币 → 消耗金币使用 Normal Hint / Wind Hint / Rotate
  ├── 没金币 → 看广告获得免费Hint（位置2）
  ├── 没金币 → 看广告获得金币（位置3）→ 再买道具
  └── 不想看广告 → 去商城花钱买金币包
```

**本质：道具系统是广告和付费的入口。** 所有道具只提供便利不影响胜负——没有道具也一定能过关。但道具让过关更快更爽，制造"想用但不够"的需求，驱动广告观看和付费。

### 7.3 MVP道具（位置1-3 + 位置5-6）

| 道具 | 功能 | 价格 | 获取方式 |
|------|------|------|----------|
| Bonus Words 查看 | 查看已找到的Bonus Words和进度条 | 免费 | — |
| Free Hint | 免费获得一次Normal Hint | 免费（看广告） | 激励视频 |
| Get Coins | 获取金币 | 免费（看广告） | 激励视频 |
| Normal Hint（灯泡） | 揭示一个目标单词的起始字母位置 | 20金币 | 金币购买/分享好友/看广告 |
| Rotate（旋转） | 整个网格旋转180度，换视角找词 | 15金币 | 金币购买/看广告 |

### 7.4 后续道具（1.1+）

| 道具 | 功能 | 价格 | 版本 |
|------|------|------|------|
| Wind Hint（风提示） | 移除网格中所有不属于目标单词的干扰字母 | 200金币 | 1.1 |

MVP有3个消耗道具（Normal Hint + Rotate + Free Hint广告），1.1加入Wind Hint后共4个。道具体系三级力度：
- **轻度**：Normal Hint — 揭示一个词的起始位置，20金币
- **换视角**：Rotate — 旋转网格180度，15金币
- **重度**：Wind Hint — 清除所有干扰字母，200金币

不再额外增加道具。放大镜功能与Normal Hint重叠，发音功能已内置在学习模式中（找到词自动TTS发音），无需做成付费道具。

### 7.5 设计原则

- **不影响胜负，只提供便利** — 没有道具也一定能过关，道具只是加速
- **道具 = 广告入口** — Free Hint 和 Get Coins 直接关联激励视频，Wind Hint 高定价制造金币饥渴
- **与分享绑定** — 分享好友可获得道具，作为裂变入口
- **教育产品避免强付费** — 不存在"不花钱就过不了"的设计
- **Daily Dash 轮换奖励中包含道具** — 让玩家免费体验高价道具（如Wind Hint），制造后续购买需求

## 八、积分与货币经济系统

### 8.1 学习积分（Learning Score）

- **只增不减**的累计分数，类似原版的 Wisdom Score
- 基于每关找到的**目标词数量**计算（每个目标词 +1 学习积分）
- **Bonus Word 不计入学习积分**
- **Hidden Words 不计入学习积分**
- 用于省份排行榜排名
- 不可消耗，纯荣誉/进度指标
- **驱动称号徽章升级**（见8.4）

## 九、省份排行与社交系统

### 9.1 省份自动归属（MVP）

- 通过微信小程序API获取用户地理位置
- 自动分配到对应省份，用户无需手动选择
- 数据字段预留 `province_code`、`city_code`、`school_id`

### 9.2 省份集体数据（1.2版本）

**排行维度：**
- 累计学习积分（省份总量，基于目标词数量）
- 人均学习积分（避免大省碾压小省）
- 本周进步榜（进步最大的省份）

**展示方式：**
- 主界面可见省份排名卡片
- "你的省份本周累计学了 X 个单词，全国排名第 Y"
- "你为广东省贡献了 Z 个单词"

**集体里程碑：**
- 省份达到累计目标 → 全省玩家获得奖励
- 例："广东省累计学习10万个单词，全体广东玩家获得10金币"
- 制造"一起学习"的集体感，而非"谁比谁强"的竞争感

### 9.3 分享裂变机制（1.2版本）

**核心路径：**
```
玩家需要道具 → 分享给微信好友 → 双方各得5金币
→ 好友看到游戏 → 好友也开始玩 → 好友也分享 → 裂变传播
```

**分享触发点：**
| 场景 | 分享奖励 |
|------|----------|
| 道具不足时 | 分享获得1个Hint |
| 关卡通关后 | 分享获得5金币 |
| 省份里程碑达成 | 分享庆祝，获得特殊道具 |
| 连续打卡成就 | 分享获得金币加成 |

**设计原则：**
- 分享是"可选捷径"，不是"必须操作"
- 不分享也能正常玩，分享只是加速
- 教育产品的分享话术要温和："邀请好友一起学英语"

### 9.4 好友排行（1.3版本）

- 基于微信社交链，自然形成好友排行
- 排行维度：本周学习积分
- 好友头像+称号徽章+学习积分，一目了然
- 微信小游戏原生支持好友排行榜，开发成本低

### 9.5 学习竞赛活动（1.3版本）

**工作日学习赛（类Weekday Tournament）：**
- 每个工作日一场，持续约24小时
- 独立竞赛货币"学习星"（类似原版 Wisdom Leaves），每期清零
- 通关即自动收集学习星，按学习星数量排名
- 参与门槛极低：完成1关即可进入
- 排名奖励：

| 排名 | 金币奖励 |
|------|----------|
| 第1名 | 500金币 |
| 第2名 | 250金币 |
| 第3名 | 150金币 |
| 第4-6名 | 100金币 |
| 第7名+ | 参与奖5金币 |

- **→ 产出 Crown Points**：参与工作日学习赛累积 Crown Points，推进当前赛季皇冠进度条

**周末冲分赛（类Wisdom Tournament）：**
- 周末举办，限时竞赛
- 用活动期间获得的学习积分排名（历史积分不计入）
- 奖励金币和专属徽章
- **→ 产出 Crown Points**：参与周末冲分赛累积 Crown Points，推进当前赛季皇冠进度条

**周中收集赛（类River Race）：**
- 每周特定日期开放（如周二、周五），每周2次
- 常规关卡中，特定目标词旁出现"星标"标记
- 找到带星标的词 = 收集1个星标（正常找词，零额外操作）
- 星标转化为 Frame Points，按 Frame Points 排名
- 历史星标不累计，只算当期
- **→ 产出 Frame Points**：收集星标累积 Frame Points，推进当前赛季头像框进度条

### 9.8 活动系统与收藏系统联动设计（1.3版本）

这是整个产品中期的核心留存架构。每个活动都有对应的收藏出口，每个收藏都有明确的活动来源。

**活动 → 收藏 完整链路：**

| 活动 | 活动频率 | 产出物 | 对应收藏 | 收藏周期 |
|------|----------|--------|----------|----------|
| 常规关卡通关（完成单元） | 持续 | 单元完成进度 | 景区卡片（Landmarks） | 超长线（按教材单元） |
| 每日挑战（Daily Challenge） | 每日1次 | 完成的名言/金句 | 名言收藏（Quotes） | 每日累积 |
| 工作日学习赛 + 周末冲分赛 | 每工作日+每周末 | Crown Points | 皇冠收藏（Crowns） | 季度赛季轮换 |
| 周中收集赛 | 每周二/五（2次） | Frame Points（星标→积分） | 头像框收藏（Frames） | 季度赛季轮换 |
| 主题头像活动 | ~3天一期，每周换主题 | 代币→开箱→碎片 | 头像收藏（Avatars） | 每周更新 |

- 每张卡片为精美插画风格，点击可查看中英双语景区介绍
- 时间跨度：超长线，随教材内容扩展持续增长
- 对应原版：Stamps（邮票），原版是完成一个国家解锁邮票

**② 皇冠收藏（Crowns）— 1:1复刻原版**
- 获取方式：参与工作日学习赛和周末冲分赛，累积 Crown Points
- 赛季制：按四季轮换（春 Mar-Jun / 夏 Jun-Sep / 秋 Sep-Dec / 冬 Dec-Mar）
- 进度条节点：0 → 10 → 30，每个节点解锁一个赛季限定皇冠样式
- 赛季结束积分清零，下赛季新皇冠样式
- 积分来源链路：
```
工作日学习赛（每工作日）→ 参与/排名 → +Crown Points
周末冲分赛（每周末）→ 参与/排名 → +Crown Points
→ Crown Points 在当前赛季进度条累积
→ 达到10分 → 解锁第一个皇冠
→ 达到30分 → 解锁第二个皇冠
→ 赛季结束 → 积分清零，新赛季新皇冠
```

**③ 头像框收藏（Frames）— 1:1复刻原版**
- 获取方式：参与周中收集赛，收集星标转化为 Frame Points
- 赛季制：与 Crowns 相同的四季轮换
- 进度条节点：0 → 10 → 40（比 Crowns 的30更高，因为周中收集赛每周2次频率较高）
- 赛季结束积分清零
- 积分来源链路：
```
周中收集赛（每周二/五）→ 找到带星标的目标词 → 收集星标
→ 星标转化为 Frame Points
→ Frame Points 在当前赛季进度条累积
→ 达到10分 → 解锁第一个头像框
→ 达到40分 → 解锁第二个头像框
→ 赛季结束 → 积分清零，新赛季新头像框
```

**④ 名言收藏（Quotes）— 1:1复刻原版**
- 获取方式：完成每日挑战（Daily Challenge）
- 每天一句新名言/课本金句，完成后自动收录
- 便签纸卡片样式展示，记录收集日期
- 时间跨度：每日累积，持续增长

**⑤ 头像收藏（Avatars）— 1:1复刻原版**
- 获取方式：参与限时主题头像活动
- 活动机制：通关收集代币（特定目标词旁标记）→ 每10代币开1个奖励箱 → 随机获得头像碎片 → 每个头像需3碎片合成
- 每周更换主题，每个主题8个普通头像 + 1个Premium头像（集齐全部后解锁）
- 已收集头像永久保留

## 十二、系统架构设计

### 12.1 四层架构

```
第一层：学段配置（用户选择，决定关卡起点和内容池）
第二层：关卡数据（静态，按学段组织的JSON关卡包）
第三层：活动配置（动态，服务端下发，全学段共享）
第四层：运行时合成（客户端合成关卡+活动标记）
```

| 层级 | 职责 | 数据位置 | 变化频率 |
|------|------|----------|----------|
| 学段配置 | 决定玩家从哪开始、能玩什么 | 用户数据（服务端） | 首次选择，极少变 |
| 关卡数据 | 词汇内容、网格、难度 | JSON关卡包（本地/CDN） | 发版时更新 |
| 活动配置 | 当前活动、标记规则、奖励 | 服务端下发 | 每天/每周变 |
| 运行时合成 | 关卡+活动标记合并渲染 | 客户端内存 | 每次进关卡实时计算 |

```json
        "per_mark": 1,
        "currency": "avatar_token"
      }
    }
  ]
}
```

**运行时合成流程：**

```
玩家打开游戏
  │
  ├── 读取用户配置 → stage: "primary", display_level: 35
  │
  ├── 加载关卡数据 → primary_animals/level5.json
  │   → 目标词: ["cat", "dog", "bird", "fish", "rabbit"]
  │
  ├── 请求活动配置 → 服务端返回当前活动列表
  │   → 活动A: 周中收集赛（mark_ratio: 0.4, icon: star）
  │   → 活动B: 主题头像活动（mark_ratio: 0.2, icon: compass）
  │
  ├── 运行时合成（客户端计算）
  │   → 目标词5个
  │   → 活动A标记 ceil(5×0.4)=2个，用 hash(event_A_id + level_id) 选词
  │   → 活动B标记 ceil(5×0.2)=1个，从剩余词中选
  │   → 一个词只能被一个活动标记，按优先级依次分配
  │   → 最终: cat, dog⭐, bird🧭, fish, rabbit⭐
  │
  └── 渲染界面（活动标记显示在目标词旁）
```

**random_seed 保证公平性：**
- 用 `hash(event_id + level_id)` 做随机种子
- 同一活动期 + 同一关卡 = 所有玩家看到相同的标记词
- 不同活动期 + 同一关卡 = 标记词会变（因为 event_id 变了）

**需要标记目标词的活动 vs 不需要的：**

| 活动 | 是否需要标记目标词 | 说明 |
|------|-------------------|------|
| 每日登录奖励 | 不需要 | 登录就领，和关卡无关 |
| 每日学习冲刺 | 不需要 | 通关就计数 |
| 每日挑战 | 不需要 | 独立关卡 |
| 单词大师 | 不需要 | 找词就推进路径 |
| 工作日学习赛 | 不需要 | 通关就得学习星 |
| 周末冲分赛 | 不需要 | 用学习积分排名 |
| 周中收集赛 | **需要** | 特定词旁标星标 |
| 主题头像活动 | **需要** | 特定词旁标代币 |

实际只有2类活动需要标记词，冲突概率很低。

**学段与活动的关系：**
- 活动配置全学段共享，不区分小学/初中/高中/成人
- 活动标记叠加在各自学段的关卡上（小学生的关卡标记小学词汇，高中生的关卡标记高中词汇）
- 排行榜全学段混排（方案A），所有人在同一个排行榜竞争
- 省份排行也是全学段混排，比的是集体贡献总量
