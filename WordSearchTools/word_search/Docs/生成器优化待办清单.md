# 生成器优化待办清单

> 本文档记录 Word Search 生成器在完成 **P0（布局美观）** 与 **P1（工具链）** 之后的后续优化任务。
> 所有任务均以"**只改生成器 / 关卡工具**"为前提，不依赖运行时玩法或服务端。
>
> 优先级说明：
> - P2 = 趣味性增强，生成器独立闭环，数据向后兼容
> - P3 = 视觉/主题玩法
> - P4 = 词库/选词层
> - ❌ = 依赖运行时/后端，暂不做

## 七、待策划/客户端确认的决策

以下 5 点在推进 P2/P3 之前应先对齐：

1. **JSON 字段扩展**：客户端是否愿意接受 `distractor_mode / decoyCells / gridMask / metaWord / specialCells / seed / layoutScore / difficulty_auto` 等新字段？未识别时默认忽略即可。
2. **P3-1 mask 数据源**：策划在 Excel 里画 ASCII，还是工具侧内置枚举？
3. **P2-2 假词库**：自动"改一字母"够用，还是要一个词频表？产品希望假词在什么语义范围？
4. **反向单词默认策略**：当前由 `useHardToggle` 控制（困难=含反向）。未来是否需要再拆一个"是否允许反向"的独立开关？
5. **Seed / 难度分是否回写 Excel**：若要，`ExcelLevelReader` 需改为读写。
