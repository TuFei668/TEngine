# Word Search Generator Unity版 - 功能说明

## 🎯 项目简介

Unity C#版单词搜索拼图生成器，基于Python版本完整移植并扩展了彩色答案可视化功能。

**版本**: v1.0

**开源协议**: GPL-3.0

## ✨ 核心功能

### 1. 单词拼图生成 ⭐⭐⭐⭐⭐

**功能**: 从单词列表自动生成单词搜索拼图

**算法**: 回溯法（Backtracking）
- 逐个尝试放置单词
- 遇到冲突时回溯
- 自动调整拼图尺寸
- 智能位置缓存

**配置选项**:
- **方向模式**: 简单(4方向) / 困难(8方向)
- **尺寸因子**: 1-99（默认4）
- **交叉偏好**: 避免 / 随机 / 偏好

**使用示例**:
```csharp
using WordSearchGenerator;

List<string> words = new List<string> { "HELLO", "WORLD", "UNITY" };
Generator gen = new Generator();
WordSearchData puzzle = gen.GenerateWordSearch(words);

Debug.Log(puzzle.puzzleText);
```

**输出示例**:
```
H E L L O K M N
W O R L D P Q R
U N I T Y S T V
A B C D E F G H
```

```csharp
    });
});
thread.Start();
```

## 🎯 配置选项详解

### 尺寸因子 (Size Factor)

**定义**: 拼图总字符数 = 单词字符数 × 尺寸因子

**计算公式**:
```
总字符数 = Σ(每个单词长度) × 尺寸因子
拼图边长 = √(总字符数)
```

**示例**:
```
单词: HELLO(5), WORLD(5) = 10个字母
尺寸因子 = 4
总字符 = 10 × 4 = 40
边长 = √40 ≈ 6.3 → 7

结果: 7x7拼图
```

**推荐值**:
- **1-2**: 非常紧凑（困难）
- **3-5**: 适中（推荐）⭐
- **6-10**: 宽松（简单）

### 交叉偏好 (Intersect Bias)

**Avoid (-1)**: 避免单词交叉
- 单词尽量不重叠
- 拼图更大
- 生成可能较慢

**Random (0)**: 随机放置（默认）⭐
- 不特意避免或追求交叉
- 平衡的结果
- 生成速度适中

**Prefer (1)**: 偏好单词交叉
- 单词尽量重叠
- 拼图更紧凑
- 生成更快

## 🔍 常见问题

### Q1: 生成速度慢怎么办？
**A**: 
- 减少单词数量（< 10个）
- 增大尺寸因子（4 → 6）
- 使用简单模式（4方向）
- 使用Prefer交叉偏好

### Q2: 如何确保生成成功？
**A**: 
- 确保单词都是大写字母A-Z
- 单词长度不要过长（< 20个字母）
- 使用合理的尺寸因子（3-6）

### Q3: 文件保存在哪里？
**A**: 
```csharp
Debug.Log(FileManager.GetSaveDirectoryPath());
// 输出完整路径
```

### Q4: 如何查看彩色答案？
**A**: 
- 方法1: 生成后点击"View Answers"按钮
- 方法2: 在答案查看器场景中加载谜题

### Q5: 颜色不够用怎么办？

- 系统自动使用HSV生成更多颜色
- 支持无限数量的单词
